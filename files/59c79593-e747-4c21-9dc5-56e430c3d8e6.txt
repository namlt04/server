so sánh final và const 
- cả 2 đều không thể thay đổi sau khi gán giá trị 
- thời điểm gán 
+ final : chạy chương trình
+ const : biên dịch
- Khởi tạo lại
+ final : dùng để tạo nhiều bản sao khác nhau
+ const : tái sử dụng cùng 1 instance
- thường dugn2 khi 
+ final : giá trị chỉ được biết khi chay 
+ giá trị đã biết ngay lúc viết code

runtime : khi chạy chương rình
compile-time: khi biên dịch code


kiểu dữ liệu:
dynamic : kiểu dữ liệu động, không kiểm tra thời điểm biên dịch, kiểm tra ở thời điểm chạy
Object : kiểu cơ bản nhất trong dart, tất cả class đều kế thừa từ object
var : khai báo biến mà Dart tự động suy luận kiểu dữ liệu dựa trên giá trị gán cho biến đó 

var : nếu chưa gán giá trị ( var x ) thì nó sẽ là dynamic
var : nếu đã gán giá trị ( var x = 10 ) thì nó tự động chuyển x thành int rồi

late : khai báo biến nhưng khởi tạo muộn ( lazy initialization ) 
+ không cần khởi tạo giá trị ngay khi khai báo biến
+ bảo đảm sẽ gán giá trị cho biến đó trước khi dùng
+ truy cập bien1 đó mà chưa gán giá trị, dart báo lỗi
